**Goal:** Translate all SEO/AEO content (`descriptions`, `home.json`, `tools.json`) from English into 9 locales for all 35 tools, ensuring each locale has idiomatic phrasing and CJK locales have proper `searchTerms`.

**Architecture:** Each locale is an independent task. Within each task, update `home.json`, `tools.json` (searchTerms for CJK), and all 35 tool description files (`{tool}.json`). Verify with JSON lint and `npm run build` after each locale. Commit after each locale is complete.

## Translation Source of Truth

English (`public/locales/en/`) is the source of truth. Every key that exists in English must exist in every locale. Before translating any locale, diff the English files against the target locale to find missing keys.

## Pre-flight: Diff English vs All Locales

- [ ] **Step 0: Find missing keys across all locales**

Run this script to identify which keys are missing in each locale:

```bash
for locale in zh-CN zh-TW ja ko es pt-BR fr de ru; do
  echo "=== $locale ==="
  for file in public/locales/en/*.json; do
    basename=$(basename "$file")
    target="public/locales/$locale/$basename"
    if [ ! -f "$target" ]; then
      echo "  MISSING FILE: $basename"
    else
      en_keys=$(node -e "const e=Object.keys(require('./$file'));console.log(e.sort().join('\n'))")
      loc_keys=$(node -e "const e=Object.keys(require('./$target'));console.log(e.sort().join('\n'))")
      missing=$(comm -23 <(echo "$en_keys") <(echo "$loc_keys"))
      if [ -n "$missing" ]; then
        echo "  MISSING KEYS in $basename: $missing"
      fi
    fi
  done
done
```

This output drives the actual work in Tasks 1–9. Only translate keys that are missing or different from English structure.

## Translation Guidelines for Tool Description Files

When translating the `descriptions` object within each tool's JSON file, follow these patterns:

### Key Translation Patterns

| English Key Pattern                       | Translation Approach                                                                                                                |
| ----------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| `aeoDefinition`                           | One-sentence SEO description. Start with tool name in locale, describe function, end with "runs in browser / no server" equivalent. |
| `whatIsTitle`                             | "What is X?" → locale equivalent (什么是X？ / Xとは？ / X란？ / ¿Qué es X？ / Qu'est-ce que X ? / Was ist X? / Что такое X?)        |
| `whatIsP1` / `whatIsP2` / `whatIsP3`      | Translate paragraphs preserving markdown links `[text](/route)`. Keep link paths unchanged.                                         |
| `howTitle` / `howP1` etc.                 | "How to" sections. Translate step descriptions naturally.                                                                           |
| `useCasesP1` etc.                         | Use cases. Keep specific examples (language names, tool names) in their standard form.                                              |
| `limitationsP1` etc.                      | Translate limitations as-is. Keep technical details (sizes, limits) unchanged.                                                      |
| `faq1Q` / `faq1A` etc.                    | FAQ pairs. Use conversational style. Q: short question. A: 1-3 sentence answer.                                                     |
| `stepsTitle` / `step1Title` / `step1Desc` | Step-by-step guides. Translate titles and descriptions. Keep numbered order.                                                        |
